# LaTeX 可视化公式编辑器项目规格书

## 1. 项目概述

### 1.1 项目名称（暂定）

LaTeX Visual Formula Editor

中文暂定名：LaTeX 可视化公式编辑器

### 1.2 产品定位

本项目计划开发一款类似 AxMath 的桌面数学公式编辑软件。

软件不负责完整 LaTeX 文档编译，也不集成 TeX Live。它的核心目标是让用户通过以下多种方式共同编辑同一个数学公式：

1. 在公式编辑区直接输入 LaTeX 命令，并实时转换为可视化公式结构；
2. 点击顶部公式工具栏中的分式、根号、积分、矩阵等模板，将结构插入当前光标位置；
3. 在可视化公式中直接点击、选择、删除和修改公式结构；
4. 查看、编辑并复制最终生成的标准 LaTeX 代码；
5. 使用带检索、预览、历史记忆和个性化排序的 LaTeX 命令输入系统。

软件最终输出标准 LaTeX 代码，供用户粘贴到 Overleaf、TeXstudio、VS Code、Typora、Markdown 编辑器或其他 LaTeX 编译环境中。

## 3. 核心使用方式

整个软件围绕同一个结构化公式编辑器工作。

```text
直接输入 LaTeX 命令 ─┐
点击公式工具栏       ├─> 同一个可视化公式编辑器 ─> 输出 LaTeX
鼠标直接修改公式     ┘
```

三种输入方式必须能够随时混合使用，而不是三个互相独立的编辑区域。

### 3.1 直接输入 LaTeX 命令

用户在可视化公式编辑区中直接输入：

```latex
\frac
```

确认命令后，编辑器立即生成分式结构：

```latex
\frac{□}{□}
```

光标自动进入分子位置。

其他示例：

```latex
\sqrt
\int
\sum
\alpha
\begin{matrix}
```

均应转换为对应的可视化公式结构。

### 3.2 点击公式工具栏

用户可以点击顶部公式栏中的按钮插入公式模板，例如：

- 分式；
- 根号；
- 上标；
- 下标；
- 积分；
- 求和；
- 极限；
- 括号；
- 矩阵；
- 希腊字母；
- 关系符号；
- 箭头；
- 集合符号；
- 常用物理符号。

模板必须插入到当前光标位置，并自动进入第一个可编辑占位符。

### 3.3 可视化直接编辑

用户需要能够：

- 点击分子、分母、上下标等区域；
- 使用方向键在公式结构中移动；
- 使用 Tab 在模板占位符之间移动；
- 选中公式的一部分；
- 对选中内容添加根号、括号、分式等外层结构；
- 删除单个符号或完整结构；
- 在矩阵中移动单元格；
- 增加或删除矩阵行列；
- 使用撤销和重做。

### 3.4 LaTeX 源码模式

软件需要提供可展开的 LaTeX 源码区域。

用户可以：

- 查看当前公式对应的 LaTeX；
- 复制 LaTeX；
- 粘贴已有 LaTeX；
- 修改完整 LaTeX；
- 将修改后的源码重新同步到可视化公式编辑器。

主编辑区域仍然是可视化公式编辑器，源码模式属于辅助功能。

## 4. 智能 LaTeX 命令输入系统

这是本项目区别于普通公式编辑器的核心功能之一。

### 4.1 输入法式交互

当用户输入反斜杠后，软件进入命令输入状态。

例如用户输入：

```latex
\f
```

光标附近出现候选框：

```text
1. \frac        分式
2. \forall      任意
3. \fbox        方框
4. \floor       下取整
```

继续输入：

```latex
\fr
```

候选项自动缩小：

```text
1. \frac        分式
2. \frak        花体字体
```

### 4.2 候选项内容

每个候选项应包含：

- 可视化公式预览；
- 标准 LaTeX 命令；
- 中文名称；
- 可选英文名称；
- 所属分类；
- 必要时显示使用提示。

候选框应尽量显示在当前光标附近，而不是固定在窗口侧边。

### 4.3 候选框键盘操作

- 上方向键：选择上一项；
- 下方向键：选择下一项；
- Enter：确认候选；
- Tab：确认并进入第一个占位符；
- Esc：关闭候选框；
- 鼠标点击：选择候选项；
- Cmd/Ctrl + Z：撤销插入。

### 4.4 不允许过早自动替换

用户输入：

```latex
\f
```

软件只能显示候选，不得立即自动替换为 `\frac`。

只有用户主动确认候选后，命令才能转换成公式结构。

## 5. 命令检索系统

命令检索不能只支持简单前缀匹配。

### 5.1 前缀匹配

```text
\f   -> \frac、\forall、\fbox
\sq  -> \sqrt、\square、\sqsubset
```

### 5.2 模糊匹配

用户存在少量拼写错误时仍能检索到正确命令。

```text
\frc      -> \frac
\squr     -> \sqrt
\interal  -> \int
```

### 5.3 中文检索

可以考虑支持：

```text
\分式
\积分
\矩阵
\希腊字母
```

最终插入的仍然是标准 LaTeX，而不是中文命令。

### 5.4 英文别名检索

例如：

```text
\divide    -> \frac
\root      -> \sqrt
\infinity  -> \infty
```

### 5.5 分类检索

命令至少需要分为：

- 基础结构；
- 微积分；
- 线性代数；
- 矩阵；
- 希腊字母；
- 关系符号；
- 集合与逻辑；
- 箭头；
- 运算符；
- 字体；
- 物理常用；
- 用户自定义。

## 6. 命令记忆与个性化推荐

### 6.1 目标

软件需要根据每位用户的命令使用习惯，动态调整候选项顺序。

例如用户经常输入 `\f` 后选择 `\frac`，以后输入 `\f` 时，`\frac` 应优先显示。

如果用户输入 `\fo` 后经常选择 `\forall`，则 `\forall` 应在 `\fo` 查询下优先显示。

### 6.2 需要记录的数据

每个命令建议记录：

- 总使用次数；
- 最近一次使用时间；
- 最近若干次使用时间；
- 用户在什么输入前缀下选择了该命令；
- 用户通过键盘、工具栏或历史记录中的哪种方式使用；
- 在何种公式上下文中使用；
- 是否收藏；
- 是否固定在顶部。

### 6.3 统一使用记录

以下操作都应视为使用了一次对应命令：

- 从候选框选择命令；
- 点击公式工具栏；
- 从历史公式或收藏模板中插入；
- 使用快捷键插入。

### 6.4 推荐排序因素

推荐顺序应综合考虑：

1. 当前输入与命令的匹配程度；
2. 当前前缀下的历史选择次数；
3. 命令总使用次数；
4. 最近使用时间；
5. 当前公式上下文；
6. 用户是否收藏或固定；
7. 系统默认优先级。

可使用以下概念模型：

```text
最终得分
= 匹配得分
+ 前缀历史得分
+ 使用频率得分
+ 最近使用得分
+ 上下文得分
+ 收藏或固定得分
```

### 6.5 时间衰减

不能只按照总使用次数排序。

旧的使用记录应逐渐降低影响，最近经常使用的命令应更快提升排序。

### 6.6 可控制性

设置中需要提供：

- 开启或关闭个性化推荐；
- 清除命令使用记录；
- 重置推荐顺序；
- 导出个人命令记录；
- 导入个人命令记录；
- 固定命令；
- 取消固定；
- 调整候选项数量。

- 公式解释、纠错和化简。

## 16. 核心验收标准

第一版成功的最低标准：

1. 用户可以在公式编辑框中输入 `\frac`；
2. 输入过程中出现候选框；
3. 用户选择 `\frac` 后出现分式结构；
4. 光标自动进入分子；
5. 用户可以用 Tab 进入分母；
6. 用户可以点击工具栏插入同样的分式；
7. 用户可以直接点击公式内部进行修改；
8. 软件可以实时得到标准 LaTeX；
9. 用户可以复制该 LaTeX；
10. 用户下一次输入相同前缀时，常用命令会根据历史记录提前；
11. 用户可以关闭或重置个性化排序；
12. 整个过程不依赖 TeX Live，不进行 PDF 编译。

## 17. 项目核心原则

1. 公式编辑器只有一个核心数据源：LaTeX；
2. 可视化输入、命令输入和工具栏输入必须共同编辑同一个公式；
3. 不自行重写完整数学排版和光标引擎，优先使用 MathLive；
4. 智能命令检索与个性化推荐独立开发；
5. 推荐算法必须可解释、可重置、可关闭；
6. 用户确认前不得擅自替换未完成命令；
7. 模板插入后必须自动进入符合直觉的编辑位置；
8. 第一版聚焦公式编辑和 LaTeX 输出，不扩展成完整 LaTeX IDE；
9. 所有主要功能优先保证键盘操作效率；
10. 产品体验应接近 AxMath，但突出个性化 LaTeX 命令输入法这一差异化能力。
